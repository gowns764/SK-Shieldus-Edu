구성요소 - 산출물

기획서 -> 프로젝트 방향을 정하고 어떤 것을 시작으로 어떤 것을 할 것인지 정확이 기술, 범위, 기획 의도 : 필요한 기술 어떤 것이 필요한지

요구사항 정의서 -> 기획서를 기반으로 사용자의 요구사항을 정의
-> 정의한 것을 기준으로 표 형식으로 무엇을 만들지 정의 : 프로그램 결과물 또는 화면 등의 정의를 기술한 것

일정표(WBS) -> 달력 형식으로 일정을 정의 : 요구사항 정의서에 정의된 항목을 기준으로 달력으로 표시 (간트 차트)

프로젝트 성격에 맞추어서 산출물이 조금 다르게 나옴
클라우드 네이티브 서비스 목록 / 클라우드 서비스 목록
퍼블릭 클라우드 서비스 예상 항목 + 예상 금액
보안 서비스에 충족되는 서비스 항목 + 예상 금액
프로젝트 성격에 맞는 법률 매핑 정의서 => 서비스 정의서 => 보안 요건 정의서
보안 아키텍처 정의서(명세서)
개볌, 논리, 물리적 설계서 = ACL 문서, AWS 클라우드 아키텍처, CIDR 정의서(NACL/SG), 방화벽
= ACL/WAF => Rule 정의서, 침입방지/탐지의 Rule 정의서, 인증/권한 ACL과 정책 문서
